# Drift Report Spec v1

## Purpose

`risk drift` computes deterministic drift status for a runset by comparing:
- reference baseline card from baseline discovery (`baseline resolve`)
- current baseline card generated by `risk sweep`

It is read-only with respect to baseline root and uses fixed thresholds.

## Command

```text
risk drift --catalog <catalog_dir> --root <baseline_root> --runset <runset_id> [--label prod] --out <dir>
```

## Output Layout

```text
<out>/<runset_id>/
  current/
    <runset_id>/
      baseline.card.json
      baseline.card.manifest.json
  diff/
    baseline.diff.json
    baseline.diff.manifest.json
  drift.report.json
  drift.report.manifest.json
```

## DriftReport v1

Fields:
- `drift_version`: `v1`
- `created_utc`: `RESEARCH_CREATED_UTC`
- `runset_id`
- `label`: resolved label used
- `reference`
  - `baseline_root`
  - `baseline_path` (runset-relative path under baseline root)
  - `baseline_id` (`per_run_vector_sha256`)
  - `baseline_card_canonical_sha256`
- `current`
  - `baseline_path` (path under drift output root)
  - `baseline_id` (`per_run_vector_sha256`)
  - `baseline_card_canonical_sha256`
- `diff`
  - `diff_path` (path under drift output root)
  - `diff_canonical_sha256`
  - `classification_label`
  - `instability_mean_delta`
  - `worst5_jaccard`
- `checks`
  - `per_run_vector_match`
- `drift_classification`
  - `status` in `{NO_DRIFT, DRIFT}`
  - `rationale` short stable string
- `drift_report_canonical_sha256`

## Fixed Drift Rule v1

`DRIFT` iff:
- `abs(instability_mean_delta) >= 5.0` OR
- `worst5_jaccard <= 0.40`

Else `NO_DRIFT`.

No knobs are supported in v1.

## Manifest v1

`drift.report.manifest.json` includes:
- `manifest_version: v1`
- `created_utc`
- sorted `inputs` with byte `sha256` and `bytes`
- `outputs.drift.report.json` with `sha256` and `bytes`
- `drift_manifest_canonical_sha256`

Inputs include:
- runset entry json
- dataset_to_runs index json
- reference baseline card + manifest
- current baseline card + manifest
- baseline diff json + manifest
