# Docs Style Guide

## Truth sources and non-hallucination rules

- Reference pages must be generated or copied from repository truth sources only:
	- CLI help generated by `tools/docs/gen_cli_ref.py`
	- JSON schemas generated by `tools/docs/gen_schema_ref.py`
	- Vendored spec markdown in `docs/reference/contracts/v1/`
- Tutorials must link to reference pages and contracts; do not restate CLI flags or schema fields unless cited to the relevant reference page.
- Unknown details must be recorded as `TODO:` markers; never guess flags, paths, schema keys, or behaviors.
- Canonical terms for this wiki are: `run`, `runset`, `stage`, `artifact`, `manifest`, `baseline`, `drift`, `risk`, `golden`.

## Enable repo hooks

Enable the committed repo-local hooks path once per clone:

```bash
git config core.hooksPath .githooks
```

```bash
RESEARCH_SKIP_DOCS_HOOK=1 git commit -m "..."
```

After this is enabled, the `pre-commit` hook will regenerate and stage generated docs
(`docs/reference/cli`, `docs/reference/schemas`, `docs/reference/artifacts`) when
staged changes may affect docs outputs.
